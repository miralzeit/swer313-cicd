# Payment Retry MySQL Migration Note

Payment retry support allows multiple `payment` rows for the same `booking_id`, as long as only one active `PENDING` payment exists and no `SUCCESS` payment exists.

The current JPA mapping uses `@ManyToOne` from `Payment` to `Booking`, so the application code no longer expects `payment.booking_id` to be unique. Existing MySQL databases may still have an old unique index or unique constraint from the previous one-to-one mapping. `spring.jpa.hibernate.ddl-auto=update` usually does not remove obsolete unique constraints automatically.

## Check Existing Indexes

Run this against the application database:

```sql
SHOW INDEX FROM payment WHERE Column_name = 'booking_id';
```

Look at the `Non_unique` column:

- `Non_unique = 0` means the index is unique and blocks payment retries.
- `Non_unique = 1` means the index is non-unique and does not block payment retries.

To list only unique indexes/constraints on `payment.booking_id`:

```sql
SELECT
    s.INDEX_NAME,
    s.NON_UNIQUE,
    s.COLUMN_NAME
FROM INFORMATION_SCHEMA.STATISTICS s
WHERE s.TABLE_SCHEMA = DATABASE()
  AND s.TABLE_NAME = 'payment'
  AND s.COLUMN_NAME = 'booking_id'
  AND s.NON_UNIQUE = 0;
```

## Drop Old Unique Index

Replace `<index_name>` with the unique index name returned by the query above. Common names are `UK...` generated by Hibernate, or sometimes `booking_id`.

```sql
ALTER TABLE payment DROP INDEX <index_name>;
```

Example:

```sql
ALTER TABLE payment DROP INDEX UK_payment_booking_id;
```

## Ensure A Non-Unique Index Exists

After dropping the unique index, keep a normal index for lookup performance:

```sql
CREATE INDEX idx_payment_booking_id ON payment (booking_id);
```

If an index with that name already exists, MySQL will reject the command. Check indexes first with:

```sql
SHOW INDEX FROM payment WHERE Key_name = 'idx_payment_booking_id';
```

## Verify Retry Is Unblocked

After migration, this query should return no unique index for `booking_id`:

```sql
SELECT
    s.INDEX_NAME,
    s.NON_UNIQUE,
    s.COLUMN_NAME
FROM INFORMATION_SCHEMA.STATISTICS s
WHERE s.TABLE_SCHEMA = DATABASE()
  AND s.TABLE_NAME = 'payment'
  AND s.COLUMN_NAME = 'booking_id'
  AND s.NON_UNIQUE = 0;
```
